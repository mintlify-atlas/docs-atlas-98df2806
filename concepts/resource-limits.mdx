title: Resource limits
description: Resource limits and execution boundaries to prevent runaway scripts
---

BashKit enforces strict resource limits to prevent denial-of-service attacks and runaway scripts.

## ExecutionLimits

The `ExecutionLimits` struct configures all resource boundaries:

```rust
use bashkit::ExecutionLimits;
use std::time::Duration;

let limits = ExecutionLimits::new()
    .max_commands(10_000)              // Command fuel
    .max_loop_iterations(10_000)       // Per-loop limit
    .max_total_loop_iterations(1_000_000) // Global loop limit
    .max_function_depth(100)           // Recursion limit
    .timeout(Duration::from_secs(30))  // Execution timeout
    .parser_timeout(Duration::from_secs(5))  // Parse timeout
    .max_input_bytes(10_000_000)       // 10 MB script limit
    .max_ast_depth(100)                // AST nesting limit
    .max_parser_operations(100_000);   // Parser fuel

let mut bash = Bash::builder()
    .limits(limits)
    .build();
```

## Default Limits

Default values are designed for typical workloads:

<ParamField path="max_commands" type="usize" default="10,000">
  Maximum number of commands that can be executed (fuel model)
</ParamField>

<ParamField path="max_loop_iterations" type="usize" default="10,000">
  Maximum iterations for a single loop
</ParamField>

<ParamField path="max_total_loop_iterations" type="usize" default="1,000,000">
  Maximum total loop iterations across all loops (prevents nested loop multiplication)
</ParamField>

<ParamField path="max_function_depth" type="usize" default="100">
  Maximum function call depth (recursion limit)
</ParamField>

<ParamField path="timeout" type="Duration" default="30 seconds">
  Maximum execution time for the entire script
</ParamField>

<ParamField path="parser_timeout" type="Duration" default="5 seconds">
  Maximum time allowed for parsing (separate from execution)
</ParamField>

<ParamField path="max_input_bytes" type="usize" default="10,000,000">
  Maximum input script size (10 MB)
</ParamField>

<ParamField path="max_ast_depth" type="usize" default="100">
  Maximum AST nesting depth during parsing
</ParamField>

<ParamField path="max_parser_operations" type="usize" default="100,000">
  Maximum parser operations (fuel model for parsing)
</ParamField>

## Limit Categories

### Command Fuel

**Mitigates:** TM-DOS-002, TM-DOS-004, TM-DOS-019

Each command execution decrements the command counter:

```rust
let limits = ExecutionLimits::new()
    .max_commands(5);

let mut bash = Bash::builder()
    .limits(limits)
    .build();

// This will fail after 5 commands
let result = bash.exec("
    echo 1
    echo 2
    echo 3
    echo 4
    echo 5
    echo 6  # Fails here: ResourceLimitExceeded::CommandCount
").await;
```

**What counts as a command:**
- Builtin executions (`echo`, `grep`, etc.)
- Variable assignments in command position
- Control flow constructs (counted once per iteration)

**What doesn't count:**
- Variable declarations (`FOO=bar` without execution)
- Comments
- Empty lines

### Loop Limits

**Mitigates:** TM-DOS-016, TM-DOS-017, TM-DOS-018

#### Per-Loop Limit

Each loop can iterate up to `max_loop_iterations` times:

```bash
# This loop will fail after 10,000 iterations (default)
for i in $(seq 1 20000); do
    echo $i
done
# Error: ResourceLimitExceeded::LoopIterations
```

#### Total Loop Limit

Prevents nested loop multiplication attacks:

```bash
# Inner loop: 10,000 iterations
# Outer loop: 10,000 iterations
# Total: 10,000 * 10,000 = 100,000,000 iterations
# This would exceed max_total_loop_iterations (1,000,000)

for i in $(seq 1 10000); do
    for j in $(seq 1 10000); do
        echo "$i $j"
    done
done
# Error: ResourceLimitExceeded::TotalLoopIterations
```

From `limits.rs:42-46`:
> Nested loops each reset their per-loop counter, allowing 10K^depth total iterations. This global cap prevents that.

### Function Depth

**Mitigates:** TM-DOS-020, TM-DOS-021

Limits recursion depth to prevent stack overflow:

```bash
function recurse {
    local depth=$1
    echo "Depth: $depth"
    recurse $((depth + 1))
}

recurse 0
# Error after 100 calls: ResourceLimitExceeded::FunctionDepth
```

**Mutual recursion is also limited:**

```bash
function a {
    b
}

function b {
    a
}

a
# Error after 100 total calls
```

### Timeouts

**Mitigates:** TM-DOS-023

#### Execution Timeout

Maximum time for script execution:

```rust
let limits = ExecutionLimits::new()
    .timeout(Duration::from_secs(5));

let mut bash = Bash::builder()
    .limits(limits)
    .build();

// This will timeout after 5 seconds
let result = bash.exec("
    while true; do
        sleep 0.1
    done
").await;
// Error: ResourceLimitExceeded::Timeout
```

#### Parser Timeout

**Mitigates:** TM-DOS-024

Separate timeout for parsing to prevent parser hang attacks:

```rust
let limits = ExecutionLimits::new()
    .parser_timeout(Duration::from_secs(2));

let mut bash = Bash::builder()
    .limits(limits)
    .build();

// Complex nested script that takes too long to parse
let deeply_nested = "({".repeat(10000) + "echo x" + "})" * 10000;
let result = bash.exec(&deeply_nested).await;
// Error: ResourceLimitExceeded::ParserTimeout
```

### Input Size

**Mitigates:** TM-DOS-001

Maximum script size to prevent memory exhaustion:

```rust
let limits = ExecutionLimits::new()
    .max_input_bytes(1000); // 1 KB limit

let mut bash = Bash::builder()
    .limits(limits)
    .build();

let large_script = "echo hello\n".repeat(1000);
let result = bash.exec(&large_script).await;
// Error: ResourceLimitExceeded::InputSize
```

From `limits.rs:62-64`:
> Default: 10MB (10,000,000 bytes). Protects against memory exhaustion from large scripts.

### AST Depth

**Mitigates:** TM-DOS-022

Limits nesting depth during parsing:

```bash
# Deeply nested subshells
((((((((((echo too deep))))))))))
# Error: ResourceLimitExceeded::AstDepth
```

From `limits.rs:67-69`:
> Default: 100. Protects against stack overflow from deeply nested scripts.

### Parser Operations

**Mitigates:** TM-DOS-024

Fuel model for parser to prevent CPU exhaustion:

```rust
let limits = ExecutionLimits::new()
    .max_parser_operations(1000);

let mut bash = Bash::builder()
    .limits(limits)
    .build();

// Complex script with many tokens
let complex = "echo foo; ".repeat(10000);
let result = bash.exec(&complex).await;
// Error: ResourceLimitExceeded::ParserOperations
```

From `limits.rs:72-74`:
> Default: 100,000. Protects against parser DoS attacks that could otherwise cause CPU exhaustion.

## Error Types

When limits are exceeded, you get specific error types:

```rust
use bashkit::Error;

match result {
    Err(Error::ResourceLimitExceeded(limit)) => {
        match limit {
            ResourceLimitExceeded::CommandCount => {
                println!("Too many commands executed");
            }
            ResourceLimitExceeded::LoopIterations => {
                println!("Loop exceeded iteration limit");
            }
            ResourceLimitExceeded::TotalLoopIterations => {
                println!("Total loop iterations exceeded");
            }
            ResourceLimitExceeded::FunctionDepth => {
                println!("Function recursion too deep");
            }
            ResourceLimitExceeded::Timeout => {
                println!("Execution timed out");
            }
            ResourceLimitExceeded::ParserTimeout => {
                println!("Parser timed out");
            }
            ResourceLimitExceeded::InputSize => {
                println!("Script too large");
            }
            ResourceLimitExceeded::AstDepth => {
                println!("Script nesting too deep");
            }
            ResourceLimitExceeded::ParserOperations => {
                println!("Parser operations exceeded");
            }
        }
    }
    Ok(result) => {
        println!("Success: {}", result.stdout);
    }
    Err(e) => {
        println!("Other error: {}", e);
    }
}
```

See [Error handling guide](/guides/error-handling) for more details.

## Adjusting Limits

### For Long-Running Scripts

```rust
let limits = ExecutionLimits::new()
    .max_commands(100_000)           // More commands
    .timeout(Duration::from_secs(300)); // 5 minute timeout
```

### For Short Scripts

```rust
let limits = ExecutionLimits::new()
    .max_commands(100)
    .timeout(Duration::from_secs(5));
```

### For Trusted Input

```rust
let limits = ExecutionLimits::new()
    .max_input_bytes(100_000_000)  // 100 MB
    .max_ast_depth(500)
    .max_parser_operations(1_000_000);
```

<Warning>
Increasing limits reduces protection against DoS attacks. Only adjust limits for trusted input or when you understand the security implications.
</Warning>

### Disabling Limits

You can set limits to very high values, but not disable them entirely:

```rust
let limits = ExecutionLimits::new()
    .max_commands(usize::MAX)
    .max_loop_iterations(usize::MAX)
    .timeout(Duration::from_secs(86400)); // 24 hours
```

## Per-Tool-Call Timeout

When using `BashTool`, you can override the global timeout per call:

```rust
use bashkit::{BashTool, ToolRequest};

let mut tool = BashTool::builder()
    .timeout(Duration::from_secs(30))  // Default
    .build();

let request = ToolRequest {
    commands: "long-running-command".to_string(),
    timeout_ms: Some(60_000),  // 60 second timeout for this call
};

let response = tool.execute(request).await;
```

From `CHANGELOG.md:35`:
> Per-tool-call `timeout_ms` in ToolRequest

## Monitoring Limits

Limits are checked at key points during execution:

- **Command count:** Before each command execution
- **Loop iterations:** At the start of each loop iteration
- **Function depth:** When entering a function
- **Timeout:** Periodically during execution (every ~100ms)
- **Input size:** Before parsing
- **AST depth:** During parsing
- **Parser operations:** During tokenization and parsing

## Security Mitigations

From `limits.rs:6-17`, resource limits mitigate these threat model categories:

| Threat ID | Attack | Mitigation |
|-----------|--------|------------|
| TM-DOS-001 | Large script input | `max_input_bytes` |
| TM-DOS-002, TM-DOS-004, TM-DOS-019 | Command flooding | `max_commands` |
| TM-DOS-016, TM-DOS-017 | Infinite loops | `max_loop_iterations` |
| TM-DOS-018 | Nested loop multiplication | `max_total_loop_iterations` |
| TM-DOS-020, TM-DOS-021 | Function recursion | `max_function_depth` |
| TM-DOS-022 | AST depth attack | `max_ast_depth` |
| TM-DOS-023 | CPU exhaustion | `timeout` |
| TM-DOS-024 | Parser hang | `parser_timeout`, `max_parser_operations` |
