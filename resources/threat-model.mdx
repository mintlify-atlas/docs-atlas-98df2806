This document provides detailed analysis of security threats and their mitigations.

## Overview

BashKit assumes all script input is potentially malicious. The virtual environment prevents:

- Resource exhaustion (CPU, memory, disk)
- Sandbox escape (filesystem, process, privilege)
- Information disclosure (secrets, host info)
- Network abuse (exfiltration, unauthorized access)

## Threat ID Categories

All threats use stable IDs in the format `TM-<CATEGORY>-<NUMBER>`:

| Prefix | Category |
|--------|----------|
| TM-DOS | Denial of Service |
| TM-ESC | Sandbox Escape |
| TM-INF | Information Disclosure |
| TM-INJ | Injection |
| TM-NET | Network Security |
| TM-ISO | Multi-Tenant Isolation |
| TM-INT | Internal Error Handling |
| TM-LOG | Logging Security |
| TM-GIT | Git Security |
| TM-PY | Python/Monty Security |
| TM-UNI | Unicode Security |

## Denial of Service (TM-DOS-*)

### Memory Exhaustion

| ID | Threat | Mitigation |
|----|--------|------------|
| TM-DOS-001 | Large script input | `max_input_bytes` limit (10MB) |
| TM-DOS-002 | Output flooding | Command limit stops loop |
| TM-DOS-003 | Variable explosion | No /dev/urandom in VFS |
| TM-DOS-004 | Array growth | Command limit |

### Filesystem Exhaustion

| ID | Threat | Mitigation |
|----|--------|------------|
| TM-DOS-005 | Large file creation | `max_file_size` limit (10MB) |
| TM-DOS-006 | Many small files | `max_file_count` limit (10K) |
| TM-DOS-007 | Zip bomb | Decompression limit |
| TM-DOS-008 | Tar bomb | FS limits |
| TM-DOS-009 | Recursive copy | FS limits |
| TM-DOS-010 | Append flood | FS limits + loop limit |

### Path Attacks

| ID | Threat | Mitigation |
|----|--------|------------|
| TM-DOS-011 | Symlink loops | No symlink following |
| TM-DOS-012 | Deep directory nesting | `max_path_depth` limit (100) |
| TM-DOS-013 | Long filenames | `max_filename_length` (255) |
| TM-DOS-014 | Many directory entries | `max_file_count` limit |
| TM-DOS-015 | Unicode path attacks | Path validation rejects control chars |

|----|--------|------------|

| TM-DOS-018 | Nested loops | `max_total_loop_iterations` (1M) |
| TM-DOS-019 | Command loop | Command limit (10K) |

### Stack Overflow

| ID | Threat | Mitigation |
|----|--------|------------|
| TM-DOS-020 | Function recursion | Depth limit (100) |
| TM-DOS-021 | Command sub nesting | Inherited depth budget |
| TM-DOS-022 | Parser recursion | `max_ast_depth` limit (100) |

### CPU Exhaustion

| ID | Threat | Mitigation |

## Mitigations Reference

### Configuration Example

```rust
use bashkit::{Bash, ExecutionLimits, FsLimits, InMemoryFs, NetworkAllowlist};
use std::sync::Arc;
use std::time::Duration;

// Execution limits
let limits = ExecutionLimits::new()
    .max_commands(10_000)
    .max_loop_iterations(10_000)
    .max_function_depth(100)
    .timeout(Duration::from_secs(30))
    .max_input_bytes(10_000_000);  // 10MB

// Filesystem limits
let fs_limits = FsLimits::new()
    .max_total_bytes(100_000_000)  // 100MB
    .max_file_size(10_000_000)     // 10MB per file
    .max_file_count(10_000);

let fs = Arc::new(InMemoryFs::with_limits(fs_limits));

// Network allowlist
let allowlist = NetworkAllowlist::new()
    .allow("https://api.example.com")
    .allow_domain("cdn.example.com");

// Build with all protections
let bash = Bash::builder()
    .limits(limits)
    .fs(fs)
    .network(allowlist)
    .username("sandbox")
    .hostname("bashkit-sandbox")
    .build();
```

## Security Testing

BashKit includes comprehensive security tests:

- **Threat Model Tests** - 117 tests covering all threat categories
- **Unicode Security Tests** - Byte-boundary panics, invisible chars
- **Nesting Depth Tests** - 18 tests for parser depth attacks
- **Network Security Tests** - 53 tests for HTTP attack vectors
- **Fail-Point Tests** - 14 security control injection tests
- **Fuzz Testing** - Parser and lexer fuzzing

If you discover a security vulnerability, please report it privately via GitHub Security Advisories rather than opening a public issue.
