## Overview

The `Tool` trait defines the public contract for integrating bash execution with LLMs and AI agents. All tools must implement this trait to provide introspection (schemas, documentation) and execution capabilities.

<Warning>
This trait is a **public library contract**. Breaking changes require a major version bump. See `specs/009-tool-contract.md` for the full specification.
</Warning>

## Trait Definition

```rust
#[async_trait]
pub trait Tool: Send + Sync {
    fn name(&self) -> &str;
    fn short_description(&self) -> &str;
    fn description(&self) -> String;
    fn help(&self) -> String;
    fn system_prompt(&self) -> String;
    fn input_schema(&self) -> serde_json::Value;
    fn output_schema(&self) -> serde_json::Value;
    fn version(&self) -> &str;
    async fn execute(&mut self, req: ToolRequest) -> ToolResponse;
    async fn execute_with_status(
        &mut self,
        req: ToolRequest,
        status_callback: Box<dyn FnMut(ToolStatus) + Send>,
    ) -> ToolResponse;
}
```

```rust

```

<ResponseField name="name" type="&str">

</ResponseField>

**Example:**

```rust

assert_eq!(tool.name(), "bashkit");
```

### `help()`

Return full documentation for LLMs (human-readable with examples).

```rust
fn help(&self) -> String
```

<ResponseField name="help" type="String">
  Complete documentation in man-page style format with examples and configuration
</ResponseField>

**Format:**
The help text follows Unix man-page conventions:
- NAME section
- SYNOPSIS section
- DESCRIPTION section
- BUILTINS section (for BashTool)
- INPUT/OUTPUT sections
- EXAMPLES section
- CONFIGURATION section (if custom config exists)
- NOTES section (capability hints from custom builtins)
- WARNINGS section (missing language interpreters)

**Example:**

```rust
let help = tool.help();
assert!(help.contains("BASH(1)"));
assert!(help.contains("SYNOPSIS"));
assert!(help.contains("EXAMPLES"));
```

## Execution Methods

### `execute()`

Execute the tool and return results.

```rust
async fn execute(&mut self, req: ToolRequest) -> ToolResponse
```

<ParamField path="req" type="ToolRequest" required>
  Tool execution request. See [ToolRequest](/api/tool-request).
</ParamField>

<ResponseField name="response" type="ToolResponse">
  Execution result. See [ToolResponse](/api/tool-response).
</ResponseField>

**Example:**

```rust
use bashkit::{BashTool, Tool, ToolRequest};

#[tokio::main]
async fn main() {
    let mut tool = BashTool::default();
    
    let req = ToolRequest {
        commands: "echo hello".to_string(),
        timeout_ms: None,
    };
    
    let resp = tool.execute(req).await;
    assert_eq!(resp.stdout, "hello\n");
    assert_eq!(resp.exit_code, 0);
}
```

**Timeout Handling:**
If `ToolRequest::timeout_ms` is set, execution is aborted after the specified duration and a response with `exit_code = 124` is returned (matching bash `timeout` convention).

### `execute_with_status()`

Execute with progress callbacks.

```rust
async fn execute_with_status(
    &mut self,
    req: ToolRequest,
    status_callback: Box<dyn FnMut(ToolStatus) + Send>,
) -> ToolResponse
```

<ParamField path="req" type="ToolRequest" required>
  Tool execution request
</ParamField>

<ParamField path="status_callback" type="Box<dyn FnMut(ToolStatus) + Send>" required>
  Callback for progress updates
</ParamField>

<ResponseField name="response" type="ToolResponse">
  Final execution result
</ResponseField>

**Status Phases:**
- `validate` (0%) - Request validation
- `parse` (10%) - Script parsing
- `execute` (20%) - Execution started
- `output` - Streaming output chunks
- `complete` (100%) - Execution finished

**Example:**

```rust
use bashkit::{BashTool, Tool, ToolRequest};
use std::sync::{Arc, Mutex};

#[tokio::main]
async fn main() {
    let mut tool = BashTool::default();
    let phases = Arc::new(Mutex::new(Vec::new()));
    let phases_cb = phases.clone();
    
    let req = ToolRequest::new("echo test");
    
    let resp = tool.execute_with_status(
        req,
        Box::new(move |status| {
            phases_cb.lock().unwrap().push(status.phase.clone());
        }),
    ).await;
    
    assert_eq!(resp.stdout, "test\n");
    
    let phases = phases.lock().unwrap();
    assert!(phases.contains(&"validate".to_string()));
    assert!(phases.contains(&"complete".to_string()));
}
```

## Thread Safety

The `Tool` trait requires `Send + Sync`:
- **Send**: Tool can be transferred between threads
- **Sync**: Tool can be accessed from multiple threads

This enables:
- Parallel tool execution
- Thread-safe agent architectures
- Async runtime compatibility

## Custom Tool Implementation

To create a custom tool:

```rust
use bashkit::{Tool, ToolRequest, ToolResponse, ToolStatus, async_trait};

struct MyTool;

#[async_trait]
impl Tool for MyTool {
    fn name(&self) -> &str {
        "mytool"
    }
    
    fn short_description(&self) -> &str {
        "My custom tool"
    }
    
    fn description(&self) -> String {
        "Full description of my tool".to_string()
    }
    
    fn help(&self) -> String {
        "Detailed help text".to_string()
    }
    
    fn system_prompt(&self) -> String {
        "# My Tool\n\nBrief description\n".to_string()
    }
    
    fn input_schema(&self) -> serde_json::Value {
        serde_json::json!({
            "type": "object",
            "properties": {
                "input": {"type": "string"}
            }
        })
    }
    
    fn output_schema(&self) -> serde_json::Value {
        serde_json::json!({
            "type": "object",
            "properties": {
                "output": {"type": "string"}
            }
        })
    }
    
    fn version(&self) -> &str {
        "1.0.0"
    }
    
    async fn execute(&mut self, req: ToolRequest) -> ToolResponse {
        // Custom execution logic
        ToolResponse {
            stdout: "result".to_string(),
            stderr: String::new(),
            exit_code: 0,
            error: None,
        }
    }
    
    async fn execute_with_status(
        &mut self,
        req: ToolRequest,
        mut status_callback: Box<dyn FnMut(ToolStatus) + Send>,
    ) -> ToolResponse {
        status_callback(ToolStatus::new("validate"));
        let result = self.execute(req).await;
        status_callback(ToolStatus::new("complete"));
        result
    }
}
```
