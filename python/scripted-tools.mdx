title: ScriptedTool
description: Multi-tool orchestration via bash scripts

ScriptedTool composes multiple Python callbacks into a single bash-scriptable interface. An LLM writes one bash script that pipes, loops, and branches across all registered tools.

## Overview

ScriptedTool allows you to:
- Register Python functions as bash builtin commands
- Compose multiple tools into a single orchestrated interface
- Use full bash syntax (pipes, loops, conditionals) across tools
- Provide a single, powerful tool to LLMs instead of many simple ones

## Basic Example

```python
from bashkit import ScriptedTool

def greet(params, stdin=None):
    name = params.get('name', 'world')
    return f"hello {name}"

tool = ScriptedTool("api")
tool.add_tool("greet", "Greet a user", callback=greet)

result = tool.execute_sync("greet --name Alice")
print(result.stdout)  # hello Alice
```

## Callback Signature

Callbacks must follow this signature:

```python
def my_callback(params: dict, stdin: str | None) -> str:
    """
    Args:
        params: Dictionary of --flag values from command line
        stdin: String input from pipe (None if no pipe)
    
    Returns:
        String output (sent to stdout)
    
    Raises:
        Exception: Signals command failure
    """
    pass
```

## Adding Tools

### Simple Tool

```python
from bashkit import ScriptedTool

def get_user(params, stdin=None):
    user_id = params.get('id', 1)
    return f'{{"id": {user_id}, "name": "User{user_id}"}}'

tool = ScriptedTool("api")
tool.add_tool("get_user", "Fetch user by ID", callback=get_user)

result = tool.execute_sync("get_user --id 42")
print(result.stdout)  # {"id": 42, "name": "User42"}
```

### Tool with Schema

Provide JSON Schema for parameter validation:

```python
def get_user(params, stdin=None):
    return f'{{"id": {params["id"]}, "name": "Alice"}}'

tool.add_tool(
    "get_user",
    "Fetch user by ID",
    callback=get_user,
    schema={
        "type": "object",
        "properties": {
            "id": {"type": "integer", "description": "User ID"}
        },
        "required": ["id"]
    }
)
```

### Tool with Stdin

Tools can receive input from pipes:

```python
def transform(params, stdin=None):
    if stdin is None:
        return "No input"
    return stdin.upper()

tool.add_tool("upper", "Convert to uppercase", callback=transform)

result = tool.execute_sync("echo 'hello' | upper")
print(result.stdout)  # HELLO
```

## Orchestration with Bash

The power of ScriptedTool is combining multiple tools with bash:

```python
from bashkit import ScriptedTool
import json

def get_user(params, stdin=None):
    return json.dumps({"id": 1, "name": "Alice"})

def get_posts(params, stdin=None):
    return json.dumps({"items": [
        {"title": "Post 1", "author_id": 1},
        {"title": "Post 2", "author_id": 1}
    ]})

tool = ScriptedTool("api")
tool.add_tool("get_user", "Fetch user", callback=get_user)
tool.add_tool("get_posts", "Fetch posts", callback=get_posts)

# Orchestrate with bash
result = tool.execute_sync("""
    user=$(get_user --id 1)
    posts=$(get_posts --user_id 1)
    
    echo "User: $(echo $user | jq -r '.name')"
    echo "Posts: $(echo $posts | jq '.items | length')"
""")

print(result.stdout)
# User: Alice
# Posts: 2
```

## Environment Variables

Set environment variables visible to scripts:

```python
tool = ScriptedTool("api")
tool.env("API_KEY", "secret-key-123")
tool.env("BASE_URL", "https://api.example.com")

result = tool.execute_sync("echo $API_KEY")
print(result.stdout)  # secret-key-123
```

## Resource Limits

Configure limits when creating the tool:

```python
tool = ScriptedTool(
    "api",
    short_description="API orchestrator",
    max_commands=1000,
    max_loop_iterations=10000
)
```

## Complex Example: Kubernetes Orchestrator

Here's a real-world example with multiple tools:

```python
from bashkit import ScriptedTool
import json

def get_pods(params, stdin=None):
    namespace = params.get('namespace', 'default')
    pods = [
        {"name": "web-abc", "status": "Running"},
        {"name": "api-def", "status": "Running"},
        {"name": "worker-ghi", "status": "CrashLoopBackOff"}
    ]
    return json.dumps({"items": pods})

def get_logs(params, stdin=None):
    pod = params.get('name', '')
    return f"[ERROR] Connection failed\n[ERROR] Retrying...\n"

tool = ScriptedTool("kubectl", short_description="Kubernetes CLI")

tool.add_tool(
    "get_pods",
    "List pods in namespace",
    callback=get_pods,
    schema={
        "type": "object",
        "properties": {
            "namespace": {"type": "string"}
        }
    }
)

tool.add_tool(
    "get_logs",
    "Get pod logs",
    callback=get_logs,
    schema={
        "type": "object",
        "properties": {
            "name": {"type": "string"}
        },
        "required": ["name"]
    }
)

# LLM-generated script to diagnose issues
result = tool.execute_sync("""
    # Find unhealthy pods
    unhealthy=$(get_pods --namespace default | \
        jq -r '.items[] | select(.status != "Running") | .name')
    
    # Get logs for each
    for pod in $unhealthy; do
        echo "=== Logs for $pod ==="
        get_logs --name "$pod" | grep ERROR
    done
""")

print(result.stdout)
```

## LLM Integration

ScriptedTool provides methods for LLM integration:

```python
tool = ScriptedTool("api")
tool.add_tool("get_data", "Fetch data", callback=my_callback)

# Get system prompt for LLM
print(tool.system_prompt())

# Get input schema
print(tool.input_schema())

# Get description
print(tool.description())
```

## Best Practices

### Return JSON for Structured Data

Makes it easy to pipe to `jq`:

```python
import json

def get_user(params, stdin=None):
    data = {"id": 1, "name": "Alice"}
    return json.dumps(data)
```

### Use Schema for Validation

Provide schemas for better error messages:

```python
tool.add_tool(
    "fetch",
    "Fetch resource",
    callback=fetch,
    schema={
        "type": "object",
        "properties": {
            "id": {"type": "integer"}
        },
        "required": ["id"]
    }
)
```

### Handle Stdin Gracefully

```python
def process(params, stdin=None):
    if stdin is None:
        raise ValueError("This tool requires piped input")
    return stdin.upper()
```

### Set Resource Limits

Always configure limits for LLM-generated scripts:

```python
tool = ScriptedTool(
    "api",
    max_commands=500,
    max_loop_iterations=5000
)
```
