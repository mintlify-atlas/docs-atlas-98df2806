<Warning>
Monty is an early-stage Python interpreter that may have undiscovered crash or security bugs. Resource limits are enforced by Monty's runtime. The integration should be treated as experimental.
</Warning>

BashKit embeds the [Monty](https://github.com/pydantic/monty) Python interpreter, a pure-Rust implementation of Python 3.12. Python runs entirely in-memory with configurable resource limits and no host access.

## Quick Start

Enable the `python` feature and register via builder:

```rust
use bashkit::Bash;

let mut bash = Bash::builder().python().build();

let result = bash.exec("python3 -c \"print('hello from Monty')\"").await?;
assert_eq!(result.stdout, "hello from Monty\n");
```

In `Cargo.toml`:

```toml
[dependencies]
bashkit = { version = "0.1", features = ["python"] }
```

## Why Monty

Monty provides a secure, sandboxed Python environment:

- **Pure Rust** - No CPython dependency
- **Fast startup** - Sub-microsecond initialization
- **Built-in limits** - Memory, allocations, time, recursion depth
- **No filesystem/network** - Sandbox-safe by design
- **Snapshotable** - Execution state can be captured

## Usage Patterns

### Inline Code

```bash
python3 -c "print(2 ** 10)"
# Output: 1024
```

### Expression Evaluation

When no `print()` is called, the last expression is displayed (REPL behavior):

```bash
python3 -c "2 + 2"
# Output: 4
```

### Script Files (from VFS)

```bash
cat > /tmp/script.py << 'EOF'
data = [1, 2, 3, 4, 5]
print(f"sum={sum(data)}, avg={sum(data)/len(data)}")
EOF
python3 /tmp/script.py
```

### Pipelines and Command Substitution

```bash
result=$(python3 -c "print(42 * 3)")
echo "Result: $result"

echo "print('piped')" | python3
```

### Version Check

```bash
python3 --version
python3 -V
```

## Virtual Filesystem (VFS) Bridging

Python `pathlib.Path` operations are bridged to BashKit's virtual filesystem. Files created by bash are readable from Python and vice versa.

### Bash → Python

```bash
echo "important data" > /tmp/shared.txt
python3 -c "
from pathlib import Path
content = Path('/tmp/shared.txt').read_text()
print(f'Got: {content.strip()}')
"
```

### Python → Bash

```bash
python3 -c "
from pathlib import Path
_ = Path('/tmp/result.txt').write_text('computed by python\n')
"
cat /tmp/result.txt
```

### Supported Path Operations

| Operation | Example |
|-----------|----------|
| Read text | `Path('f.txt').read_text()` |
| Read bytes | `Path('f.txt').read_bytes()` |
| Write text | `Path('f.txt').write_text('data')` |
| Write bytes | `Path('f.txt').write_bytes(b'data')` |
| Exists | `Path('f.txt').exists()` |
| Is file/dir | `Path('f.txt').is_file()`, `.is_dir()` |
| Is symlink | `Path('link').is_symlink()` |
| Mkdir | `Path('d').mkdir(parents=True, exist_ok=True)` |
| Delete | `Path('f.txt').unlink()`, `Path('d').rmdir()` |
| List dir | `list(Path('.').iterdir())` |
| Stat | `Path('f.txt').stat().st_size` |
| Rename | `Path('old').rename('new')` |
| Resolve | `Path('file').resolve()`, `.absolute()` |
| Env vars | `os.getenv('KEY')`, `os.environ` |

### Architecture

```text
Python code → Monty VM → OsCall(ReadText, path) → BashKit VFS → resume
```

Monty pauses at filesystem operations, BashKit bridges them to the VFS, then resumes execution with the result (or a Python exception like `FileNotFoundError`).

## Resource Limits

Default limits prevent runaway Python code. Customize via `PythonLimits`:

```rust
use bashkit::{Bash, PythonLimits};
use std::time::Duration;

let bash = Bash::builder()
    .python_with_limits(
        PythonLimits::default()
            .max_duration(Duration::from_secs(5))
            .max_memory(16 * 1024 * 1024)   // 16 MB
            .max_allocations(100_000)
            .max_recursion(50)
    )
    .build();
```

| Limit | Default | Builder Method | Purpose |
|-------|---------|----------------|----------|
| Allocations | 1,000,000 | `.max_allocations(n)` | Prevent memory exhaustion |
| Duration | 30 seconds | `.max_duration(d)` | Prevent infinite loops |
| Memory | 64 MB | `.max_memory(bytes)` | Heap memory cap |
| Recursion | 200 | `.max_recursion(depth)` | Call stack depth limit |

## Python Feature Support

Monty implements a subset of Python 3.12:

### Supported

- Variables, assignments, augmented assignments (`+=`, `-=`, etc.)
- Arithmetic, comparison, logical operators
- Control flow: `if`/`elif`/`else`, `for`, `while`, `break`, `continue`
- Functions: `def`, `return`, default args, `*args`, `**kwargs`
- Star unpacking in calls: `func(*args, **kwargs)`
- Data structures: `list`, `dict`, `tuple`, `set`, `frozenset`, `namedtuple`
- List/dict/set comprehensions, generator expressions
- String operations, f-strings
- Exception handling: `try`/`except`/`finally`/`raise`
- Property descriptors (`@property`)
- Built-in functions: `print`, `len`, `range`, `enumerate`, `zip`, `map`, `filter`, `sorted`, `reversed`, `sum`, `min`, `max`, `abs`, `round`, `int`, `float`, `str`, `bool`, `list`, `dict`, `tuple`, `set`, `type`, `isinstance`, `hasattr`, `getattr`, `id`, `repr`, `ord`, `chr`, `hex`, `oct`, `bin`, `all`, `any`, `input`
- Standard modules: `sys`, `typing`, `os`, `pathlib`
- `TYPE_CHECKING` guard support

### Not Supported

<Warning>
The following Python features are not available in Monty:
</Warning>

- **Classes** - Planned upstream
- **Match statements** - Not implemented
- **`open()` builtin** - Use `pathlib.Path` instead
- **HTTP/network I/O** - No `socket`, `urllib`, `requests`, `http.client`
- **Third-party imports** - No `pip install`, no `import numpy`
- **Most standard library modules** - Only `sys`, `typing`, `os`, `pathlib` available
- **`str.format()`** - Use f-strings instead

## Workarounds

### File I/O Without `open()`

Monty does not implement Python's `open()`. Use `pathlib.Path` instead:

```python
# Won't work:
# f = open('data.txt')
# content = f.read()

# Use instead:
from pathlib import Path
content = Path('data.txt').read_text()

# Writing:
Path('output.txt').write_text('data')
```

## LLM Tool Integration

When using `BashTool` for AI agents, Python limitations are automatically documented:

```rust
use bashkit::BashTool;

let tool = BashTool::builder()
    .python()
    .build();

// help() and system_prompt() automatically document Python limitations
let help = tool.help();  // Includes NOTES section with Python hints
```

The builtin's `llm_hint()` is automatically included:

> python/python3: Embedded Python (Monty). File I/O via pathlib.Path only (no open()). No HTTP/network. No classes. No third-party imports.

## Security

All Python execution runs in a virtual environment:

- **No host filesystem access** - All paths resolve through the VFS
- **No network access** - No sockets, HTTP, or DNS
- **No process spawning** - No `os.system()`, `subprocess`, or shell access
- **Resource limited** - Allocation, time, memory, and recursion caps
- **Path traversal safe** - `../..` is resolved by VFS path normalization

### Threat Mitigations

| Threat | Mitigation |
|--------|------------|
| Code injection via bash variables | Use single quotes to prevent expansion |
| Resource exhaustion | Monty enforces independent limits |
| Sandbox escape via filesystem | All paths go through VFS |
| Sandbox escape via os/subprocess | Monty has no `os.system()` or `subprocess` |
| Denial of service via large output | 64 MB memory limit prevents unbounded output |
| Path traversal | VFS path normalization constrains `../..` |

See `specs/006-threat-model.md` (TM-PY-*) for detailed threat analysis.

## Error Handling

Python errors are returned with appropriate exit codes:

| Error Type | Exit Code | Behavior |
|------------|-----------|----------|
| Syntax errors | 1 | Python traceback on stderr |
| Runtime errors | 1 | Python traceback on stderr, stdout preserved |
| File not found | 2 | Error on stderr |
| Missing `-c` argument | 2 | Error on stderr |
| Unknown option | 2 | Error on stderr |

## Integration with BashKit

Python works seamlessly with bash features:

- **Both names work**: `python` and `python3` map to the same builtin
- **Pipelines**: `echo "data" | python3 -c "import sys; ..."`
- **Command substitution**: `result=$(python3 -c "print(42)")`
- **Conditionals**: `if python3 -c "1/0"; then ... else ... fi`
- **Shebang lines**: `#!/usr/bin/env python3` are stripped automatically
